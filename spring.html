Spring DI comparison

Limitations relative to Spring DI

  • Kanuka only has Singleton scope
  • Kanuka does not have an equivalent to @Value
  • Kanuka does not have integrated AOP

Relative benefits

  • Kanuka uses Java annotation processing to generate Java source code to perform dependency injection (like Dagger)
  • Kanuka does not use classpath scanning (Spring users might use spring-component-indexer to avoid this)
  • Kanuka determines the ordering of dependency injection at compile time and not runtime
  • Kanuka does not use reflection

Kanuka performs DI very fast using compiled code avoiding reflection and classpath scanning.
